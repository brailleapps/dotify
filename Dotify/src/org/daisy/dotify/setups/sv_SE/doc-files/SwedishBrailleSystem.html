Overview

This document applies to the following Conversion Setups:

  • Swedish - A4
  • Swedish - FA44

All of them produce uncontracted Swedish braille.

Swedish - A4

This setup will create documents intended for A4 while maintaining compatibility with existing software. This means that the paper size in the output will be narrower than A4, i.e. 32 braille cells wide. Other properties of this setup are:

  • Page height: 29 characters
  • Inner margin: 2 characters
  • Outer margin: 2 characters
  • Row gap: 0
  • Volume size: about 50 sheets

Swedish - FA44

This setup will create documents intended for FA44 while maintaining compatibility with existing software. This means that the paper size in the output will be narrower than FA44, i.e. 42 braille cells wide. Other properties of this setup are:

  • Page height: 29 characters
  • Inner margin: 3 characters
  • Outer margin: 3 characters
  • Row gap: 0
  • Volume size: about 50 sheets

Input Requirements

A DTBook document. Notes and tables are not allowed.

Output

A PEF-document containing one or serveral volumes.

Configuration

A number of properties can be overridden by supplying a Java xml properties file to the conversion via the optional parameter "Settings file". The file should contain one or more of the following entries:

rows
Set the number of rows, i.e. the height, of the page (integer > 0)
cols
Set the number of columns, i.e. the width, of the page (integer > 0)
inner-margin
Set the inner margin, in characters (integer >= 0)
outer-margin
Set the outer margin, in characters (integer >= 0)
rowgap
Set the row gap (integer >= 0)
max
Set the volume splitter max parameter, see below
l10nrearjacketcopy
Set the text string to be used when announcing this particular block of text
l10nimagedescription
Set the text string to be used when announcing this particular block of text
l10ncolophon
Set the text string to be used when announcing this particular block of text
l10ncaption
Set the text string to be used when announcing this particular block of text
finalizer-input
Set the braille finalizer finalizer-input parameter, see below
finalizer-output
Set the braille finalizer finalizer-output parameter, see below

Volume Splitter

The volume splitter used in this setup is very simple. It will distribute pages into volumes equally with a maximum of "max" pages per volume.

Braille Finalizer

The Braille finalizer performs post rendering character replacing. The purpose is to replace remaining non braille characters that are needed in the layout process, such as space, no-break space, hyphen and soft hyphen. It takes two parameters: finalizer-input and finalizer-output.

For each row, the braille-finalizer replaces occurrences of characters in the finalizer-input string with the character at the corresponding position in the finalizer-output string.
