Provides a braille translation system. The system can convert structural XML documents into braille (or other text-oriented display formats, i.e. formats that have a fixed width in characters per row).

Adding a new locale

Adding a new locale requires some work, depending on your requirements. At least the following items need to be considered:

  • Add a new package in "org.daisy.dotify.setups", for example "de_DE". All locale specific code should be stored here except text filters, see Adding a language below.
  • Add your setup name to the "Setup" enum in TaskSystemFactory.
  • Create a class that implements TaskSystem, and add to it the tasks that your locale needs, see below.
  • Add your TaskSystem to TaskSystemFactory. Add the lines needed to the method called "newTaskSystem".
  • Add a language, see below.

Implementing TaskSystem

A TaskSystem implementation should contain at least these steps:

  • An input format to OBFL format conversion, for example:

    setup.add(new XsltTask("DTBook to OBFL converter", dtbook2obflXsltUrl, null, xsltProps));

  • A OBFL format to a PagedMedia conversion, for example:

    setup.add(new LayoutEngineTask("OBFL to PEF converter", flow, paginator, paged));

Refer to the sv_SE implementation for more information.

Adding a language

Braille tables are a part of the "org.daisy.dotify.translator" package. The purpose of this is to allow all locales to access each others rules, and optionally output braille using that locale, even when using a setup in another language.

  • Create a table that follows the rules of UCharReplacer (for more information, see the int_daisy_unicodeTranscoder documentation in Daisy Pipeline), and place it in a new package in the "org.daisy.dotify.translator" package, e.g "de_DE/de_DE.xml".
    If your locale has special requirements (such as contractions), this method cannot be used. In this case, another implementation must be created. If your class implements the StringFilter interface, the next step can still be applied.
  • Add your locale to the system, by adding the name of your implementation to the org.daisy.dotify.translator.BrailleFilter file in META-INF/services.

Customizing layout

Page layout, such as headers and footers, as well as the interpretation of elements in the input document are customized by writing a XSLT stylesheet for the input format, so that the input document's contents is expressed using the OBFL grammar. See "flow.dtd" in the "org.daisy.dotify.formatter" package for a description of this format.

If there exist an XSLT for the input format, it could be extended as defined by XSLT. As a general guideline, it is ok to extend a stylesheet in "setups.common". It is however, not recommended to extend another locale's stylesheets. In this case, it is better to copy the stylesheet into your locale's directory, and modify it.

Adding input formats

Similarly to customizing layout, a new input format requires a translation between the grammar in the input document and OBFL. This is achieved by applying an XSLT tailored for the task.

Detecting the input format can require some work. However, the InputDetectorTaskSystem is specifically designed to aid the process of selecting and executing the correct validation rules and XSLT stylesheet for a given input document and locale. It can be inserted at the appropriate place in another TaskSystem.

Add to InputDetectorTaskSystem the lines needed to support the new format, e.g:

if (rootNS.equals("http://www.daisy.org/z3986/2005/dtbook/") && rootElement.equals("dtbook")) {inputformat = "dtbook.properties";}

Choose a suitable name for the properties file, typically using the name of the format, e.g. "html.properties".

Refer to the Javadoc for more information on how to customize the InputDetectorTaskSystem for your locale.

Adding output formats

First, consider if your needs could be met by converting the PEF file to your desired format using (or extending) Braille Utils. If not, add an output format by implementing "PagedMediaWriter". Put your creation in the "org.daisy.dotify.formatter.writers" package. If your output format requirements cannot be met by implementing "PagedMediaWriter", please consider submitting a feature request.
